## 設計系統

### 配色（CSS 自訂屬性）

```css
/* 深色主題（預設）*/
--bg:        #0d1117;   /* 主背景 */
--ocean:     #161b22;   /* 卡片背景 */
--cloud:     #e6edf3;   /* 主文字 */
--fog:       #8b949e;   /* 次要文字 */
--sky:       #9dafff;   /* 主色（品牌藍）*/
--foam:      #a8e6cf;   /* 強調綠 */
--sand:      #deac80;   /* 暖色 */
--blush:     #e0b5d4;   /* 粉 */
--lavender:  #ab72f9;   /* 紫 */
--teal:      #58c2b0;   /* 青 */
--mist:      rgba(255,255,255,0.06);  /* 邊框 */
```

404 頁面也已引入此變數系統（`:root` 定義），確保品牌一致性。

### 字體

- **主文字**：`Noto Sans TC`, 400/500/700/800
- **標題裝飾**：`Noto Serif TC`, 700

### 動畫

- 滾動進場：`.fade-in`, `.slide-up`, `.zoom-in`, `.slide-left`, `.reveal-up`
- IntersectionObserver 驅動，觸發後加 `.visible`
- 3 秒安全閘：防止動畫未觸發時元素永久隱藏
- **歷史館特殊**：`.tl-item` 元素也加了 `.fade-in` class，由同一個 observer 監聽

### Cache Busting

- 全站統一版本號：`main.js?v=XXXXXXXX`、`style.css?v=XXXXXXXX`
- 目前版本：`260412i`
- **重要**：修改 JS/CSS 後，必須同步更新所有頁面的版本號
- 建議改用 GitHub Actions 自動注入 commit hash

### HTML 結構注意事項

- 每個 `.tl-item` 必須有 `.fade-in` class（否則不會觸發進場動畫）
- `</div>` 關閉標籤必須嚴格對應：`tl-card` → `tl-item` → `timeline-era`
- 改動時建議用 Python 腳本驗證 div 平衡性（見下方範例）

```python
# 驗證各 era 的 div 平衡
with open('歷史館.html') as f:
    content = f.read()
for era in ['前傳', '創立期', '成長期', '穩定期']:
    s = content.find(era)
    e = content.find('rules-meta', s)
    section = content[s:e]
    diff = section.count('<div') - section.count('</div>')
    print(f'{era}: {"✅" if diff==0 else "❌ diff="+str(diff)}')
```

### 數值設計規範

#### 階梯限制（相對原版滿附魔）
| 屬性 | 低階 | 中階 | 高階 | 天花板上限 |
|------|------|------|------|-----------|
| 攻擊傷害 | 原版 80% | 原版 100% | 原版 115% | 不得超過 130% |
| 挖掘速度 | 原版 80% | 原版 100% | 原版 110% | 不得超過 120% |
| 防禦值 | 原版 80% | 原版 100% | 原版 110% | 不得超過 120% |
| 附魔等級 | ≤ 原版上限 | ≤ 原版上限 | ≤ 原版+1 | 絕對不得超過原版+2 |
| 特殊效果 | 無 | +50% 以內 | +100% 以內 | 不得超過 +150% |

#### 其他規範
- 數值必須實測驗證，不可直接套用原版公式
- 需評估捲軸、烽火台、套裝組合的連動效果
- 分流二不支援 MMOItems，所有特殊裝備不可使用
- **兌換經濟平衡**：兌換材料取得途徑和速率需納入考量，優先使用受控產出途徑（任務幣、活動獎勵），避免可被大量刷取的開放材料（如獄髓）

### 審核流程
1. **提案**：設計師提出方案，附完整數值表、取得方式、對比現存裝備、連動風險、調整預案
2. **審核**：至少一位非設計者的管理人員 review
3. **測試**：測試環境驗證實際效果
4. **上線**：發佈公告說明設計理念

### 調整原則
- 先觀察至少 1 週再評估
- 每次調整幅度不超過 20%
- 優先調整取得門檻，不直接砍已發放物品
- 調整須發佈正式公告（改了什麼、為什麼改、影響範圍）
- 因數據問題下架的，玩家可 100% 退回兌換資源
